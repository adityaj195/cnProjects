A srep-by-step walk through of your first HTML page

HTML is short for HyperText Markup Language.Basically, it is the "code" behind every webpage-even this one. If you're just beginning to learn HTML, let us tell you that it's a fairly easy task.HTML,without styling, can't do anything more than setting a layout, drawing a table, or creating frames - but it is handy as it helps you to structure the content correctly, which is important whn you sit down to add style to your HTML.

However simple this might seen, it is mighty useful tool when it comes to full-fledged web development. Various tools easily eliminate the HTML coding from your work process- but if you want to be in full control of your web-page, you'll need to have some command over HTML.
Through this article, we aim to give you the essential HTML building blocks that'll help you to get up and running. Reading this, you'll be able to understand an HTML source code and even modify it for your won good!

Step One - TAGS

Tags are what youl'll see the most when you look HTML souce code. A tag can ideally be seen as a wrapper to any item on your HTML document. Tags tell what magic is to be done on the content enclosed by them.
Let's look at two types of tags:

  1. I need a closing tag
  2. I don't need a closing tag

In the first example, the sentence is wrapped by two tags. The first one is called the opening tag and seconf one is called is closing tag. Everything in between is affected by the properties of the tag. Very commonly used wxamples of such tags are < html >, < head >, < body >, < strong > etc.

The second example of tags about loner tages - as in, they don't need a closing tag to function. Although it's not required, these of tags are ofthen written as < tags/ > to make the debugging of code easier. common examples of such tags are < hr/ > - used for horizontal line, < br/ >- to break the line, etc.

Step Two - HTML, HEAD, and BODY: The three pillars of your document

These tags are essential for any HTML document. They parcel out the signficant parts of your HTML code.

  1. < BODY /BODY > is palced below your < HEAD > tag, and everything that you want to be displayed on your screen comes under this tag. Text, images, links and pretty much anything you can see in your browser live inside this tag.
  2. < HTML /HTML > wraps your entire code. Everything else in your HTML document needs to come inside these tags.
  3. < HEAD /HEAD > includes things like title, styles, and scripts. Head is usually present at the top(hah!), just inside the < /HTML > tag.

Step three- A few tags that'll make your page pretty

now that you know how to step up the skelonton of your document, let's proceed with the things that will go inside your < BODY > tag and do some magic!

  1. < b /b > makes your text look bold
  2. < i /i > makes your write in cursive
  3. < u /u > underlines what you just wrote
For example, this piece of code

< html >

    < head /head >

    < body >

       < i > I am italics! < /i > < br/ >
        < b > I am bold! < b > < br/ >
        < u > And me, well. I'm underlined! < /u > < br/ >

    < /body >

< /html >

Sould produce something like this on your browser: Dont't fret too much about the < br/ >. It's just for breaking line so that you can start from the next line. Enter key does little when it comes to changing linesin your HTML document.

Tags to help you structure your content:

  1. < br/ > breaks the line, making you countinue to the next line
  2. < p > stands for paragraph. It divides your content into paragraphs

Note: You need to use these tags as sapce and enter keys do very .ittle when it commes to formatting content inside an HTML document.

Heading Tags:

Html provide you with six tags, from < H1 /H1 > to < H6 /H6 > to help you create different sized headers quickly.

Inserting an Image:

All that's good, but what fun without images on webpage? Don't worry, < IMG > to the rescue! The image tag has a mandatory attribute called "source". Basically, it tells the browser where it should look for the image. The syntax goes something like:

< img scr="path_to_your_image"/>

Furthermore, it also has attributes like height and width that let you specify the height and width you want your image to take.

Lists:

HTML has two types of lists- orderd and unorderd. Each time of your list has to be enclosed in a tag. The syntax for creating a list is fairly simple.

Supoose you want to create a list like

  1. Item1
  2. Item2
  3. Item3

The Following code will easily do the job for you:

< ul >
    < li Item1 /li >
    < li Item2 /li >
    < li Item3 /li >
< ul >



This, by the way, was an example of an unorderd list. For an ordered list, all you need to do is replace < ul > with < ol > and < ol > and < /ul > with < /ol >

Let's see what the following code does:

< html >

  < head /head >

    < body >

      < ul >
        < li I am unorderd list's item 1! /li />
        < li I am unorderd list's item 2! /li />
        < li I am unorderd list's item 3! /li />
        < li I am unorderd list's item 4! /li />
    </ul>
      < ol >
        < li I am ordered list's item1! />
        < li I am ordered list's item2! />
        < li I am ordered list's item3! />
        < li I am ordered list's item4! />
      < /ol >

    < /body >

< /html >

All of theese tags, when arranged coherently, will provide you with simple webpage consisting of images, headings, and lists. Further, there are various tags that HTML supports, and we thoroughly recommend you to check them out and play with them!
